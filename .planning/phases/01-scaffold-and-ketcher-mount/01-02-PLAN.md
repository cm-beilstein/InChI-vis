Create `src/App.tsx`. The file has two sections: module-level provider instantiation (before the component function), then the component itself.

**Critical rules (all from locked decisions):**
- `StandaloneStructServiceProvider` is instantiated at MODULE scope, outside any function — per D-13. Creating it inside the component re-creates the WASM worker on every render.
- `ketcherRef` uses `useRef`, NOT `useState` — per D-15. Putting it in state triggers unnecessary re-renders.
- `<Editor>` must NEVER be conditionally rendered — per D-09. The loading overlay sits on top, not in place of it.
- Stub `<div>` class names must match exactly what `styles.css` defines: `inchi-section`, `mapping`, `explain`, `footnote` — per D-05. Later phases replace these divs and rely on the class names.

```typescript
import { useState, useRef } from 'react';
import { StandaloneStructServiceProvider } from 'ketcher-standalone';
import type { Ketcher } from 'ketcher-core';
import { Header } from './components/Header';
import { KetcherPanel } from './components/KetcherPanel';

// Module-level — created once for the page lifetime. NEVER move inside a component.
// (D-13: provider inside component re-creates WASM worker on every render)
const structServiceProvider = new StandaloneStructServiceProvider();

export default function App() {
  const [isReady, setIsReady] = useState(false);
  // useRef, not useState — storing in state triggers unnecessary re-renders (D-15)
  const ketcherRef = useRef<Ketcher | null>(null);

  const handleInit = (ketcher: Ketcher) => {
    ketcherRef.current = ketcher;
    setIsReady(true);
  };

  return (
    <div className="app">
      <Header />
      <KetcherPanel
        isReady={isReady}
        onInit={handleInit}
        structServiceProvider={structServiceProvider}
      />
      {/* Phase 1 stubs — class names match styles.css exactly. Later phases replace with real components. */}
      <div className="inchi-section" />
      <div className="mapping" />
      <div className="explain" />
      <div className="footnote" />
    </div>
  );
}
```

Export `structServiceProvider` so `KetcherPanel` can import it:
```typescript
export { structServiceProvider };
```

Alternatively, pass it as a prop to `KetcherPanel` (shown above). Either approach is acceptable — prop passing is slightly cleaner. Use the prop approach as shown.

**Step 3 — Create `src/components/KetcherPanel.tsx`:**

This component renders the real Ketcher `<Editor>` permanently (never conditionally) with a loading overlay on top while `!isReady`. Per D-08, D-09, D-10.

- `<Editor>` is ALWAYS in the DOM regardless of `isReady` — the overlay is a sibling, not a replacement.
- `staticResourcesUrl={import.meta.env.BASE_URL}` resolves to `/explain-that-inchi/` in production and `/` in dev — both correct for WASM asset resolution (D-10).
- The overlay text "Loading editor…" uses Unicode ellipsis U+2026, not three periods — per UI-SPEC.md Copywriting Contract.
- `errorHandler` logs to console (not silent) so any Ketcher init failures are visible.

```typescript
import { Editor } from 'ketcher-react';
import type { Ketcher } from 'ketcher-core';
import type { StandaloneStructServiceProvider } from 'ketcher-standalone';
import styles from './KetcherPanel.module.css';

interface KetcherPanelProps {
  isReady: boolean;
  onInit: (ketcher: Ketcher) => void;
  structServiceProvider: InstanceType<typeof StandaloneStructServiceProvider>;
}

export function KetcherPanel({ isReady, onInit, structServiceProvider }: KetcherPanelProps) {
  return (
    <section>
      <div className="section-label">
        <span>Ketcher · sketch panel</span>
        <span className="hint">Draw a molecule to see its InChI</span>
      </div>
      <div className={styles.ketcher}>
        {/* Editor is ALWAYS rendered — never conditional. Removing and remounting
            causes WASM to re-initialize from scratch. (D-09) */}
        <Editor
          structServiceProvider={structServiceProvider}
          staticResourcesUrl={import.meta.env.BASE_URL}
          onInit={onInit}
          errorHandler={(msg) => console.error('Ketcher error:', msg)}
        />
        {/* Loading overlay sits position:absolute on top of the mounted Editor.
            Removed from DOM (not hidden) when isReady becomes true. (D-08) */}
        {!isReady && (
          <div className={styles.loadingOverlay}>
            Loading editor…
          </div>
        )}
      </div>
    </section>
  );
}
```

**Step 4 — Create `src/components/KetcherPanel.module.css`:**

The `.ketcher` class is defined in `styles.css` globally, but the KetcherPanel needs two CSS Module-scoped classes:
1. `.ketcher` — overrides/extends the global `.ketcher` for the real Editor container (single-column, since Ketcher renders its own toolbar and mol-list internally)
2. `.loadingOverlay` — positioned overlay over the Editor area

Also, THIS is where the Ketcher CSS import must go, inside the `@layer ketcher-reset` directive (per D-07). Importing it here (rather than in main.tsx) means it only loads with the KetcherPanel component, and the `@layer` ensures it loses the cascade against unlayered tokens in `styles.css`.

```css
@layer ketcher-reset {
  @import 'ketcher-react/dist/index.css';
}

.ketcher {
  /* Override the 3-column grid from global styles.css — the real Ketcher Editor
     fills the full container and renders its own toolbar/mol-list internally */
  display: block;
  background: var(--bg-canvas);
  border: 1px solid var(--line);
  border-radius: 6px;
  overflow: hidden;
  min-height: 380px;
  position: relative;
}

.loadingOverlay {
  position: absolute;
  inset: 0;
  background: oklch(0.985 0.005 85 / 0.85);
  display: flex;
  align-items: center;
  justify-content: center;
  color: var(--ink-soft);
  font-family: var(--font-mono);
  font-size: 13px;
  letter-spacing: 0.04em;
  z-index: 10;
}
```

Note on loading overlay background: `oklch(0.985 0.005 85 / 0.85)` is `--bg` at 85% opacity. CSS variables cannot be used inside the alpha slot of oklch() directly, so the literal value is used here — matches exactly what is in RESEARCH.md Pattern 3 and UI-SPEC.md Loading Overlay table.

Note on `StructServiceProvider` type: if TypeScript cannot resolve `InstanceType<typeof StandaloneStructServiceProvider>` cleanly, use `import type { StructServiceProvider } from 'ketcher-core'` instead and type the prop as `structServiceProvider: StructServiceProvider`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser console → ketcher API | DevTools console calls `ketcher.getInchi()` and `highlights.create` — these are development-time only, not a production attack surface |
| React component → Ketcher WASM | `onInit` callback receives the Ketcher instance; it is stored in a ref, not exposed globally in production |
| Ketcher CSS → project design tokens | Ketcher's `:root` declarations could override project tokens if not isolated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Ketcher CSS overriding design tokens | mitigate | `@layer ketcher-reset { @import 'ketcher-react/dist/index.css' }` in KetcherPanel.module.css. Unlayered `:root` declarations in styles.css win the cascade against layered Ketcher CSS. Verified in RESEARCH.md CSS Isolation Analysis — no variable name conflicts, but @layer is belt-and-suspenders. |
| T-02-02 | Information Disclosure | ketcherRef exposed on window | accept | ketcherRef is a React ref stored on the App component — not exposed to `window`. No PII or secrets in the Ketcher instance for Phase 1. Future phases should avoid `window.ketcher` patterns. |
| T-02-03 | Denial of Service | WASM re-initialization from conditional Editor render | mitigate | `<Editor>` is always in the DOM (D-09). The loading overlay is rendered conditionally on top of the Editor, not in place of it. This is enforced in the acceptance criteria: `{isReady && <Editor` pattern is explicitly forbidden. |
| T-02-04 | Elevation of Privilege | StandaloneStructServiceProvider inside component | mitigate | Provider instantiated at module scope (D-13), outside the App function. Acceptance criteria verify `const structServiceProvider = new StandaloneStructServiceProvider()` appears outside the component boundary. |
</threat_model>
