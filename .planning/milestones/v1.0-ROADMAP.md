## Overview

Full browser-based InChI explainer: Ketcher molecule editor (WASM, no backend) with live InChI generation, color-coded interactive layer strip, explanation cards, legend, atom-mapping strip, preset molecules, and multi-layer Ketcher canvas highlighting — all deployed as a static build to GitHub Pages.

## Milestone Summary

**Key Accomplishments:**
1. Vite + React 18 + TypeScript + Ketcher WASM scaffold with design tokens, deployed to GitHub Pages via GitHub Actions CD
2. Live InChI pipeline: debounced generation, layer parsing, AuxInfo atom mapping (canonical→Ketcher index)
3. Color-coded interactive InChI strip with explanation cards, per-layer legend, and idle/hover state management
4. Full highlight integration: layer hover + sub-token hover drives Ketcher canvas atom/bond highlights (TDD throughout, 135+ tests)
5. Multi-fragment molecule support with correct per-fragment canonical offset fix in parseAuxMapping and enrichLayers
6. Per-group h-layer sub-tokens with SVG hydrogen badges for implicit H, explicit H atom+bond highlights, mobile-H group highlights

**Requirement Coverage:** 20/20 mapped and shipped (5 requirements — INCHI-05 through INCHI-08, PLSH-04 — were added during development; REQUIREMENTS.md was not updated to reflect them)

**MAP-03 Deferred:** Shareable URL (hash/query param encoding) deferred to v2 by decision in STATE.md

**Key Decisions:**
- `@vitejs/plugin-react` (esbuild), NOT SWC — SWC crashes on Ketcher packages (issue #5565)
- All three Ketcher packages pinned to exactly 3.12.0
- `StandaloneStructServiceProvider` created at module level, never inside a component
- `vite-plugin-static-copy` v4 + `assetsInlineLimit: 0` for WASM asset serving
- Separate `vitest.config.ts` for Vite 8 + Vitest 3 (Plugin type conflict if merged)
- CSS Modules + CSS custom properties — preserves oklch token system from design handoff
- `getInchi(true)` returns concatenated string; split on `AuxInfo=`, not destructuring
- Stale closures in `editor.subscribe` avoided by reading state through `useRef` in handler

**Technical Debt / Known Issues:**
- Phase 8 badge positioning was tweaked but not fully browser-verified before phase close
- b-layer highlighting + legend hover adding Ketcher highlight (last commit) — not fully verified in browser
- REQUIREMENTS.md was never updated after day 1 — archive reflects the actual shipped state
