    Create .github/workflows/deploy.yml with the following exact content (per D-09, D-10, RESEARCH.md Pattern 3):

    ```yaml
    name: Deploy to GitHub Pages

    on:
      push:
        branches:
          - master

    permissions:
      contents: write

    jobs:
      deploy:
        runs-on: ubuntu-latest
        concurrency:
          group: ${{ github.workflow }}-${{ github.ref }}
          cancel-in-progress: true
        steps:
          - uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Build
            run: npm run build

          - name: Deploy to GitHub Pages
            uses: peaceiris/actions-gh-pages@v4
            with:
              github_token: ${{ secrets.GITHUB_TOKEN }}
              publish_dir: ./dist
    ```

    Key decisions per RESEARCH.md:
    - Branch: master (not main — D-03 in RESEARCH.md Pitfall 3)
    - Node: '20' (LTS, safe for Vite 8 + TypeScript 5)
    - `npm ci` not `npm install` (reproducible installs from package-lock.json)
    - `peaceiris/actions-gh-pages@v4` not actions/deploy-pages (simpler single-job pattern)
    - `permissions: contents: write` required by peaceiris/actions-gh-pages to push to gh-pages branch
    - `cancel-in-progress: true` prevents stale deploys from queuing
    - No `BASE_URL` env var needed — vite.config.ts already has base: '/explain-that-inchi/'

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → npm registry | npm ci fetches packages from registry during CI build |
| GitHub Actions → gh-pages branch | peaceiris/actions-gh-pages writes to the repository |
| Browser → GitHub Pages CDN | Static file delivery; WASM/SharedArrayBuffer requires COOP/COEP headers via coi-serviceworker |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Tampering | .github/workflows/deploy.yml | mitigate | Use pinned action versions (actions/checkout@v4, peaceiris/actions-gh-pages@v4) to prevent supply-chain substitution. GITHUB_TOKEN scope is write:contents only — no broader permissions granted. |
| T-06-03-02 | Spoofing | npm ci in CI | accept | npm ci uses package-lock.json checksums; lockfile is committed. Dependency confusion risk is low for a public educational tool with no secrets in the build. |
| T-06-03-03 | Information Disclosure | GITHUB_TOKEN in deploy step | accept | GITHUB_TOKEN is scoped to the repository, auto-expires after the job, and is not logged. Standard GitHub Actions pattern. |
| T-06-03-04 | Denial of Service | cancel-in-progress: true | mitigate | Concurrency cancel-in-progress prevents deploy queue pile-up from rapid pushes. Already included in workflow. |
| T-06-03-05 | Elevation of Privilege | coi-serviceworker.js injecting headers | accept | The service worker only adds COOP/COEP headers to same-origin requests; it cannot inject into cross-origin requests. Already audited in Phase 1 (public/coi-serviceworker.js v0.1.7). |
</threat_model>
