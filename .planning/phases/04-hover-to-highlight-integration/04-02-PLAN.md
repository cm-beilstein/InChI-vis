    Implement `useKetcherHighlights(ketcherRef, isReady)`:
    ```typescript
    export function useKetcherHighlights(
      ketcherRef: React.RefObject<Ketcher | null>,
      isReady: boolean
    ): void {
      const hoverIdx    = useInchiStore(s => s.hoverIdx);
      const subHover    = useInchiStore(s => s.subHover);
      const layers      = useInchiStore(s => s.layers);
      const auxMap      = useInchiStore(s => s.auxMap);
      const atomElements = useInchiStore(s => s.atomElements);

      useEffect(() => {
        if (!isReady || !ketcherRef.current) return;
        const editor = ketcherRef.current.editor;
        const struct  = (editor as any).render.ctab.molecule as StructLike;

        // Always clear first — prevents stale highlight accumulation (D-04)
        // Also clears on hoverIdx=null (idle) and non-spatial layers (D-01)
        if (hoverIdx === null) {
          editor.highlights.clear();
          return;
        }
        const layer = layers[hoverIdx];
        if (!layer) {
          editor.highlights.clear();
          return;
        }
        // Non-spatial layers: clear canvas, update explanation card only (D-01)
        if (['version', 'q', 'p', 'i'].includes(layer.type)) {
          editor.highlights.clear();
          return;
        }

        const specs = buildHighlightSpecs(layer, subHover, auxMap, atomElements, layers, struct, resolveVar);
        applyKetcherHighlights(editor, specs);
      }, [hoverIdx, subHover, layers, auxMap, atomElements, isReady]);
      // Note: ketcherRef is a ref — intentionally not in deps (stable reference)
    }
    ```

    Note: `(editor as any).render.ctab.molecule` is required because Ketcher's TypeScript types do not expose `render` on the `Editor` type. This is verified as safe — see RESEARCH.md Pattern 2.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zustand store → Ketcher API | Store values (hoverIdx, layers, auxMap, atomElements) are passed to Ketcher's highlight API |
| CSS cascade → Ketcher renderer | Resolved CSS custom property strings are passed as color values to highlights.create() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | `buildHighlightSpecs` atom IDs | mitigate | All canonical indices are mapped through `auxMap` with `.filter(id => id !== undefined)`. Ketcher's `getValidInputOnly()` silently drops any ID not in the struct pool — double defense. |
| T-04-02-02 | Tampering | `resolveVar` color strings | accept | Input is always a hard-coded CSS custom property name (e.g., `'--c-formula'`) from `elementColor()` / `hydroColor()` / `parityColor()` — never user-supplied text. No injection risk. |
| T-04-02-03 | Denial of Service | `struct.bonds.forEach` iteration | accept | Iterates over the Ketcher molecule's internal bond pool (bounded by drawn structure, max hundreds). No unbounded growth path. |
| T-04-02-04 | Elevation of Privilege | `(editor as any).render.ctab.molecule` | accept | This is a private Ketcher internal API accessed via `any` cast. Risk is API breakage on Ketcher version bump, not a security threat. Ketcher 3.12.0 is pinned; documented in STATE.md research flags. |
</threat_model>
