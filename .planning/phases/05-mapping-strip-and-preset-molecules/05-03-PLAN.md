handleMolSelect contract (D-02, D-04, RESEARCH.md Pattern 3):
```typescript
const handleMolSelect = async (id: string) => {
  const mol = MOLECULES.find(m => m.id === id);
  if (!mol || !ketcherRef.current) return;
  setIsLoading(true);
  setSelectedMolId(id);
  isSettingMoleculeRef.current = true;
  try {
    const res = await fetch(
      `https://pubchem.ncbi.nlm.nih.gov/rest/pug/compound/cid/${mol.cid}/SDF`
    );
    if (!res.ok) throw new Error(`PubChem ${res.status}`);
    const sdf = await res.text();
    await ketcherRef.current.setMolecule(sdf);
    // Do NOT call getInchi() here — setMolecule() triggers editor 'change' which fires
    // handleChange automatically. Calling getInchi() directly creates a race condition.
  } catch (err) {
    console.error('Preset load failed:', err);
    setSelectedMolId(null); // revert active state on error (D-04)
  } finally {
    isSettingMoleculeRef.current = false;
    setIsLoading(false);
  }
};
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App.tsx → PubChem HTTPS | Outbound fetch to external REST API. Response is SDF text passed to Ketcher. |
| PubChem SDF → ketcher.setMolecule() | Ketcher parses and validates SDF internally. SDF content never rendered as HTML. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Spoofing | handleMolSelect | accept | CIDs are hardcoded constants from MOLECULES array — user cannot inject arbitrary CIDs. No URL parameter or user text input feeds the PubChem URL. |
| T-05-03-02 | Tampering | PubChem response | accept | Public educational tool with no authentication. A compromised PubChem response would result in incorrect molecule display — acceptable risk for a chemistry learning tool with no auth. |
| T-05-03-03 | Denial of Service | handleMolSelect | mitigate | `disabled={isLoading}` + `isSettingMoleculeRef` prevents concurrent fetch launches. Only one fetch can be in-flight at a time. |
| T-05-03-04 | Information Disclosure | App.tsx state | accept | selectedMolId and formula are visible in React DevTools — this is public chemistry data, no sensitive information. |
| T-05-03-05 | Elevation of Privilege | setMolecule() | accept | Ketcher WASM executes in browser sandbox. SDF is parsed by Ketcher's built-in parser — no raw string is eval'd or injected into DOM as HTML. ASVS V5 (input validation) satisfied by Ketcher's internal parsing. |
</threat_model>
