<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → static host (GitHub Pages) | App served from static hosting with no server-side processing |
| browser → Google Fonts CDN | Font stylesheet loaded cross-origin; blocked by COEP without CORP headers |
| browser → WASM worker | Ketcher WASM worker runs in a separate thread; requires SharedArrayBuffer (COOP/COEP context) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | coi-serviceworker.js download | mitigate | Download from the canonical GitHub source (gzuidhof/coi-serviceworker master) at scaffold time; pin by committing the file to the repo so it cannot be replaced by a supply-chain attack later. Content reviewed at download time. |
| T-01-02 | Spoofing | Google Fonts CDN `<link>` | accept | Fonts are decorative; a compromised CDN response degrades typography but cannot execute code (stylesheets, not scripts). COEP may block the request entirely, causing fallback to system fonts — acceptable. |
| T-01-03 | Information Disclosure | SharedArrayBuffer via COOP/COEP | mitigate | coi-serviceworker applies `Cross-Origin-Opener-Policy: same-origin` and `Cross-Origin-Embedder-Policy: require-corp`, enabling the SharedArrayBuffer isolation required by spec. Without this, WASM cannot run on GitHub Pages. |
| T-01-04 | Denial of Service | WASM 404 in production | mitigate | `assetsInlineLimit: 0` prevents base64 inlining; `viteStaticCopy` copies binaryWasm assets to dist root. Both are required for WASM assets to be fetchable at their known URL. Verified in RESEARCH.md Pitfall 2. |
| T-01-05 | Elevation of Privilege | `@vitejs/plugin-react-swc` supply chain | mitigate | Use only `@vitejs/plugin-react` (esbuild). SWC plugin is explicitly excluded. Locked in D-12 and enforced by package.json devDependencies — the SWC package is not installed. |
</threat_model>
