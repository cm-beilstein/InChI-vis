## Layout Contract

Phase 1 establishes the full layout shell. These measurements are final for all phases.

**Primary focal point:** The H1 title "Explain that InChI" at 52px IBM Plex Serif draws the eye first before the editor panel — it is the largest, heaviest element on the page and anchors the visual hierarchy.

### App Shell

```
max-width: 1240px
margin: 0 auto
padding: 40px 32px 80px
```

### Component Stacking Order (top to bottom)

1. `<Header>` — flex row, space-between, bottom border, margin-bottom 28px
2. `<KetcherPanel>` — section with `.section-label` + `.ketcher` grid
3. `<div className="inchi-section">` — empty stub (margin-top 36px)
4. `<div className="mapping">` — empty stub
5. `<div className="explain">` — empty stub
6. `<div className="footnote">` — empty stub

### KetcherPanel Grid

```css
.ketcher {
  display: grid;
  grid-template-columns: 56px 1fr 240px;   /* toolbar | canvas | mol-list */
  background: var(--bg-canvas);
  border: 1px solid var(--line);
  border-radius: 6px;
  overflow: hidden;
  min-height: 380px;
}
```

Phase 1 renders the real `<Editor>` component in the center column. The 56px toolbar column and 240px mol-list column are rendered by Ketcher's own UI — the `.ketcher` div wraps the `<Editor>` which fills the full grid. The loading overlay sits `position: absolute; inset: 0` over the entire `.ketcher` container.

### Responsive Breakpoint

```css
@media (max-width: 900px) {
  .ketcher { grid-template-columns: 56px 1fr; }
  .mol-list { display: none; }
  .explain { grid-template-columns: 1fr; }
  .header h1 { font-size: 36px; }
}
```

Copy verbatim from `styles.css`. Phase 1 must include this breakpoint even though mol-list and explain are stubs.

## Interaction Contract

### Loading State (PLSH-02) — the only interaction in Phase 1

| State | Trigger | Visual |
|-------|---------|--------|
| Loading (initial) | Page load / Ketcher WASM initializing | Overlay visible: semi-transparent `--bg` at 85% opacity, "Loading editor…" centered in `--ink-soft` `--font-mono` 13px |
| Ready | `onInit` callback fires (sets `isReady = true`) | Overlay removed; Ketcher editor fully interactive |

**Overlay behavior:**
- `<Editor>` is ALWAYS rendered (never conditionally hidden — see CONTEXT.md D-09)
- Overlay is a sibling element with `position: absolute; inset: 0; z-index: 10`
- Overlay is conditionally rendered (`{!isReady && <div ...>}`) — React removes it from DOM when ready
- No transition animation on overlay removal (snap remove is acceptable for Phase 1)

### No Other Interactions in Phase 1

Phase 1 stubs are empty `<div>`s with correct class names. They have no hover states, click handlers, or interactive behavior. All interaction contracts for InChI hover, layer highlighting, and atom mapping are defined in their respective phase UI-SPECs (Phases 3, 4, 5).

## CSS Architecture

| Rule | Rationale |
|------|-----------|
| `src/styles.css` imported once in `main.tsx` | D-06 — makes all CSS custom properties globally available |
| Ketcher CSS imported inside `@layer ketcher-reset { }` | D-07 — prevents Ketcher's `:root` overrides from winning cascade |
| No Tailwind classes | CLAUDE.md — Tailwind incompatible with oklch token system |
| No inline styles except loading overlay | Inline styles only in KetcherPanel overlay (position: absolute, inset: 0) and dynamic accent colors (Phase 3+) |
| CSS Modules for component-specific layout | Use `.module.css` files only when a component needs styles not already in `styles.css` |
| Do not redefine CSS custom properties | All tokens live in `styles.css :root`; never redeclare in module files |
