## Component Inventory

Three new components are introduced in this phase. Each gets its own `.module.css` file following the established project pattern.

### 1. `MappingStrip` — `src/components/MappingStrip.tsx`

**Purpose:** Read-only display strip. Reads `auxMap` and `atomElements` from Zustand store. Renders Ketcher-rank → canonical-index pairs.

**Rendered structure (CSS class names from `src/styles.css` global classes):**

```
<div class="mapping" aria-label="Ketcher to InChI atom mapping">
  <span class="mapping-label">Ketcher → InChI</span>
  <span class="pairs">
    {pairs.map(p =>
      <span class={"pair" + (p.k === p.c ? " identity" : " diverges")}
            title="Atom drawn {k}th in Ketcher is canonical #{c} in InChI">
        <span class="k">{p.k}</span>
        <span class="arrow">→</span>
        <span class="c">{p.c}</span>
        <span class="el">{p.el}</span>
      </span>
    )}
    {!anyDiverges && (
      <span style="color: var(--ink-faint); font-style: italic; margin-left: 4px">
        (identity — drawing order already matches the canonical numbering)
      </span>
    )}
  </span>
</div>
```

**Data derivation (from CONTEXT.md D-07):**
1. `const poolIds = Object.values(auxMap).sort((a, b) => a - b)` — sorted pool IDs = draw order
2. For each canonical atom `c`: `ketcherRank = poolIds.indexOf(auxMap[c]) + 1`
3. Sort pairs by `ketcherRank` ascending

**States:**
- **Populated:** `Object.keys(auxMap).length > 0` — renders pair chips
- **Empty canvas:** `Object.keys(auxMap).length === 0` — renders nothing (component returns `null`)
- **Identity:** `ketcherRank === c` — `.identity` class, `opacity: 0.55`
- **Divergent:** `ketcherRank !== c` — `.diverges` class, green border + `--c-conn-bg` background

**Pair chip `title` tooltip text pattern:** `"Atom drawn {k}th in Ketcher is canonical #{c} in InChI"` — use this exact pattern (adapted from design handoff line 342).

**Note:** The `→` arrow in `.mapping-label` uses a styled `<span style={{ color: "var(--ink-faint)" }}>` wrapper (matching design handoff exactly — not a plain text arrow).

Source: `design_handoff_explain_that_inchi/app.jsx` lines 319–357, `src/styles.css` lines 459–508.

### 3. `KetcherPanel` additions — `src/components/KetcherPanel.tsx` (existing component, extended)

**Purpose:** Add mol-list sidebar and canvas overlay to the existing KetcherPanel wrapper. The `<Editor>` component itself is unchanged.

**New props added to KetcherPanel:**

| Prop | Type | Purpose |
|------|------|---------|
| `selectedMolId` | `string \| null` | Highlights active preset in mol-list |
| `onMolSelect` | `(id: string) => void` | Preset click handler |
| `molName` | `string \| null` | Preset molecule name for overlay (null = user-drawn) |
| `formula` | `string \| null` | Molecular formula from `layers[0].text` (null = empty canvas) |
| `heavyAtomCount` | `number` | `Object.keys(atomElements).length` |
| `isLoading` | `boolean` | True while PubChem fetch is in-flight — disables all preset buttons |

**Canvas overlay `.canvas-meta` — rendered when `formula !== null`:**

```
<div class="canvas-meta">
  <span class="dot"></span>
  {molName && <span><b>{molName}</b></span>}
  <span>{formula}</span>
  <span>· {heavyAtomCount} heavy atom{heavyAtomCount === 1 ? "" : "s"}</span>
</div>
```

- Overlay is hidden entirely when `formula === null` (empty canvas: `layers.length === 0`)
- Preset active: shows name + formula + count
- User-drawn (no preset selected): shows formula + count only — no `<b>name</b>` span rendered

**Mol-list sidebar (added inside `.ketcher` grid, after `.canvas-wrap`):**

The `KetcherPanel.module.css` already defines `display: block` overriding the 3-column grid. The mol-list sidebar must be accommodated by switching the CSS Module wrapper to a `display: grid; grid-template-columns: 1fr 240px` layout (matching the design handoff 3-column grid minus the toolbar column which Ketcher renders internally). Apply this only to the inner wrapper layout, not the Ketcher `<Editor>` itself.

```
<div class="mol-list">
  <div class="mol-list-header">Examples</div>
  {MOLECULES.map(m => (
    <button
      key={m.id}
      class={"mol-item" + (selectedMolId === m.id ? " active" : "")}
      onClick={() => !isLoading && onMolSelect(m.id)}
      disabled={isLoading}
    >
      <span class="mol-name">{m.name}</span>
      <span class="mol-formula">{m.formula}</span>
    </button>
  ))}
</div>
```

**Loading state during PubChem fetch:**
- All `.mol-item` buttons gain `disabled` attribute and `cursor: not-allowed` (via `button:disabled` CSS)
- The active item shows a spinner in place of the `.dot` indicator (or opacity 0.6 on the list — either is acceptable; use opacity 0.6 for simplicity)
- No modal or toast — fetch failure is silent (console.error only, per CONTEXT.md D-04)

**Responsive breakpoint (from `src/styles.css` line 692):**
```css
@media (max-width: 900px) {
  .mol-list { display: none; }
}
```
The mol-list is hidden on narrow viewports. This is pre-existing behaviour from the design system — do not override it.

Source: `design_handoff_explain_that_inchi/app.jsx` lines 52–101, `src/styles.css` lines 190–223, 306–350.

## Interaction Contracts

### Preset molecule click

| Event | Handler | Effect |
|-------|---------|--------|
| Click `.mol-item` (not loading) | `handleMolSelect(id)` in App.tsx | Set `selectedMolId = id`; fetch PubChem SDF; call `ketcherRef.current.setMolecule(sdf)`; existing `handleChange` fires automatically |
| Click `.mol-item` (loading) | blocked — button is `disabled` | No effect; cursor: not-allowed |
| PubChem fetch success | async in `handleMolSelect` | `setMolecule()` triggers editor change event → full pipeline updates (layers, auxMap, mapping strip) within one debounce cycle |
| PubChem fetch failure | `console.error(...)` | Leave canvas unchanged; reset `isLoading = false`; `selectedMolId` reverts to previous value |
| User draws on canvas | editor 'change' fires | `selectedMolId` is set to `null` — the canvas is no longer a known preset |

### Mapping strip hover / tooltip

| Element | Interaction |
|---------|-------------|
| `.pair` chip | `title` attribute provides browser tooltip on hover — no custom tooltip needed |
| `.pair.identity` | Dimmed (opacity 0.55); no interaction beyond tooltip |
| `.pair.diverges` | Green border + background; no click action; tooltip explains the mapping |

### Canvas overlay visibility

| State | Overlay |
|-------|---------|
| Empty canvas (`layers.length === 0`) | Hidden (`null` render) |
| Non-empty canvas, no preset selected | Shows formula + heavy-atom count; no name |
| Non-empty canvas, preset selected | Shows name (bold) + formula + count |

### Transitions

All transitions use values already declared in `src/styles.css`:
- `.mol-item` background transition: `150ms` (from `src/styles.css` line 335)
- `.pair` border-color and background transition: `160ms` (from `src/styles.css` line 496)
- Canvas `.dot` pulse animation: `2.4s ease-in-out infinite` (from `src/styles.css` line 221)

No new transition values are introduced in Phase 5.
