  <name>Task 2: Visual UAT — hover-to-highlight end-to-end</name>
  <what-built>
    The complete hover-to-highlight integration: hovering InChI layer chunks and sub-tokens now drives atom and bond highlight calls on the Ketcher canvas.

    What was wired: `useKetcherHighlights(ketcherRef, isReady)` in App.tsx reads `hoverIdx` and `subHover` from the Zustand store (set by InchiSection.tsx and LayerText.tsx mouse events) and translates them to `editor.highlights.clear()` + `editor.highlights.create()` calls on the Ketcher canvas.
  </what-built>
  <how-to-verify>
    Run `npm run dev` and open http://localhost:5173

    Draw a test molecule with at least 6 atoms — recommended: benzene (draw a 6-carbon ring) or aspirin (gives formula, c, h, and t layers).

    **Test 1 — Formula layer (INCHI-03):**
    - Hover the formula chunk (e.g. `C₆H₆`) in the InChI strip
    - Expected: Each carbon atom in the canvas gets a colored halo in its element color (`--c-formula` / `--c-el-C`)
    - Expected: Other InChI chunks dim to opacity ~0.35
    - Move cursor off → all halos clear, canvas returns to default

    **Test 2 — Connection layer (INCHI-03):**
    - Hover the `c` connection layer chunk
    - Expected: Bonds between connected atoms are highlighted in the connection color (`--c-conn`, a green-teal tone)
    - Expected: Atom endpoints of those bonds also get halos

    **Test 3 — Hydrogen layer (INCHI-03):**
    - Hover the `h` hydrogen layer chunk (benzene: `h/1-6H`)
    - Expected: All 6 carbon atoms get halos in the H-count color (`--c-hydro-1`, a warm amber)

    **Test 4 — Non-spatial layers (D-01):**
    - Draw an ionized molecule (e.g. add a charge) or use a molecule with an `s` stereo flag
    - Hover the `q` (charge) layer if present
    - Expected: Explanation card updates, canvas shows NO highlight change

    **Test 5 — Sub-token hover (INCHI-04):**
    - In the formula layer span, hover an individual element symbol (e.g. the `C` part)
    - Expected: Only the carbon atoms get halos; other element atoms (if any) remain unhighlighted
    - Expected: Layer-wide formula halo is fully suppressed (only the hovered element highlighted)

    **Test 6 — Stale highlight cleanup (INCHI-03 criterion 4):**
    - Rapidly move cursor between 3-4 different layers
    - Expected: Each layer shows only its own highlight — no accumulation of halos from previous layers
    - Expected: No atom appears doubly-highlighted from a prior hover

    **Test 7 — Idle state:**
    - Move cursor completely off the InChI strip
    - Expected: All canvas halos clear; explanation card shows "Hover any layer" default

    **Test 8 — Stereo layer (if molecule has stereocenters):**
    - Draw a chiral molecule (e.g., L-alanine, add one stereocentre)
    - Hover the `t` layer
    - Expected: Stereocentre atom(s) highlighted in `--c-stereo-plus` or `--c-stereo-minus` colors
  </how-to-verify>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React component → Ketcher internal API | `ketcherRef.current.editor.highlights` is called with integer atom/bond IDs derived from parsed InChI data |
| Store state → canvas render | Hover state in Zustand (hoverIdx, subHover) triggers side-effectful Ketcher API calls in a useEffect |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Denial of Service | `useEffect` in useKetcherHighlights | mitigate | `isReady` guard prevents API calls before Ketcher WASM is initialized. `ketcherRef.current` null-check prevents calls when ref is unset. Effect cleanup runs on unmount. |
| T-04-03-02 | Tampering | Atom/bond IDs passed to highlights.create | accept | IDs derive from `auxMap` (set by `parseInchiWithAux` from WASM output) and `struct.findBondId` (Ketcher internal). All are integers bounded by the current molecule size. Ketcher's `getValidInputOnly` silently filters any out-of-range ID. |
| T-04-03-03 | Information Disclosure | `(window as any).ketcher` in App.tsx | accept | This debugging shortcut was present in App.tsx before Phase 4 (dev-mode DX). Not a new surface. Acceptable for a public single-page educational tool with no authentication or PII. |
</threat_model>
