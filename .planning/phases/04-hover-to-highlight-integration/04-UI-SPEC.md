## Interaction Contract

This section is the core of the Phase 4 design contract. No new visual components are added; all interaction behavior is expressed as state → canvas API mappings.

### Hover State Machine

```
Idle (hoverIdx = null, subHover = null)
  → no highlights on canvas
  → explanation card shows DEFAULT_INFO

Layer hover (hoverIdx = i, subHover = null)
  → layer-wide highlight (see Layer Highlight Rules below)
  → explanation card shows LAYER_INFO[layer.type]
  → other InChI chunks dim to opacity 0.35 (`.inchi-layer.dim`)

Sub-token hover (hoverIdx = i, subHover = {...})
  → layer-wide highlight FULLY SUPPRESSED
  → only sub-token targeted atoms/bonds highlighted (strong mode)
  → explanation card unchanged (still shows layer)

Mouse leave
  → hoverIdx = null, subHover = null
  → all highlights cleared from canvas
  → canvas returns to default appearance
```

Source: ROADMAP Phase 4 success criteria 1–4; CONTEXT.md D-05; canvas.jsx lines 98–115.

### Layer Highlight Rules

| Layer Type | Canvas Effect | Color Token | Notes |
|------------|---------------|-------------|-------|
| `formula` | Halo ALL heavy atoms | per `elementColor(atom.el)` → `var(--c-el-{El})` | All atoms get their element color halo; standard opacity (.on) |
| `c` | Highlight connection bonds + endpoint atoms | `var(--c-conn)` | Uses `layer.bonds[]` canonical pairs → Ketcher bond IDs; atoms get conn color halo; standard opacity |
| `h` | Halo atoms per H-count shading | `hydroColor(count)` → `var(--c-hydro-{1|2|3|4})` or `var(--c-hydro-mobile)` | Uses `layer.atoms[]` and H-count from `hMap`; mobile H atoms get `--c-hydro-mobile` |
| `t` | Halo stereo-center atoms by parity | `parityColor(sign)` → `var(--c-stereo-{plus|minus})` | Uses `parseStereoParities(layer.text)`; strong opacity (.on-strong); also tints wedge bonds |
| `b` | Halo double-bond stereo atoms by parity | `parityColor(sign)` → `var(--c-stereo-{plus|minus})` | Strong opacity; tints bond-adjacent atoms |
| `m` | Halo all stereo-center atoms | `var(--c-stereo)` | Strong opacity |
| `s` | Halo all stereo-center atoms | `var(--c-stereo)` | Strong opacity |
| `i` | Halo isotope-labeled atoms | `var(--c-isotope)` | Standard opacity |
| `version` | No canvas change | — | Explanation card updates; canvas untouched (D-01) |
| `q` | No canvas change | — | Explanation card updates; canvas untouched (D-01) |
| `p` | No canvas change | — | Explanation card updates; canvas untouched (D-01) |

Source: canvas.jsx lines 82–115; CONTEXT.md D-01, D-02, D-03.

### Sub-token Highlight Rules

When `subHover` is non-null, the layer-wide highlight is fully suppressed. Only atoms/bonds matched by the sub-token receive a highlight. All halos use `.on-strong` (fill-opacity 0.22, stroke-opacity 0.55).

| `subHover.kind` | Targeted atoms | Color |
|-----------------|----------------|-------|
| `element` | Atoms where `atomElements[canonical] === subHover.el` | `elementColor(subHover.el)` |
| `atom` | Atom where canonical === `subHover.canonical`; incident bonds | `var(--c-conn)` |
| `stereo` | Atom where canonical === `subHover.atom` | `parityColor(subHover.sign)` |
| `hAtoms` | Atoms in `subHover.atoms[]` | `hydroColor(subHover.count)` |
| `mobileH` | Atoms in `subHover.atoms[]` | `var(--c-hydro-mobile)` |

Source: canvas.jsx `subHaloFor()` lines 22–38; CONTEXT.md D-05.

### Stale Highlight Cleanup

Prefer atomic replace (single `highlights.create` call replacing previous state) to avoid flicker. If the Ketcher 3.12.0 API only supports `clear()` + `create()`, use that as the fallback — a single-frame flicker is acceptable per D-04.

Clear-on-leave: when `hoverIdx` transitions to `null`, call the appropriate `highlights.clear()` or pass an empty highlight set to return the canvas to its default appearance.

Source: CONTEXT.md D-04. Research flag from STATE.md: "confirm sequential calls accumulate correctly in 3.12.0".

### Canvas Atom Number Display

When the `c` (connection) layer is hovered, canonical atom numbers replace Ketcher draw-order numbers. All other layers hide both number sets. This is controlled by the `showCanonical` and `hideAllNums` flags already defined in canvas.jsx — the Phase 4 Ketcher implementation must replicate this using the highlights API (not SVG class names).

Source: canvas.jsx lines 83–85; `src/styles.css` `.canonical-num` / `.ketcher-num` / `.molecule.hide-nums` classes.
