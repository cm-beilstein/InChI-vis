## Interaction Contract

### InChI strip hover states

**Trigger:** `onMouseEnter` on an `.inchi-layer` span calls `setHover(i)` and `setSubHover(null)`.
**Trigger:** `onMouseLeave` on the `.inchi-display` container calls `setHover(null)` and `setSubHover(null)`.

| State | Visual |
|-------|--------|
| Idle (hoverIdx === null) | All layers at full opacity; no backgrounds; default layer accent color text |
| Active layer (hoverIdx === i) | Layer gets `.active` class → background: `var(--c-{type}-bg)` at `160ms ease`; text color stays at layer accent |
| Dimmed layers (hoverIdx !== null, hoverIdx !== i) | Layer gets `.dim` class → `opacity: 0.35` at `160ms ease` |

**Sub-token hover:**
- Within an `.active` layer, individual sub-token spans call `setSubHover(hit)` on `onMouseEnter` and `setSubHover(null)` on `onMouseLeave`.
- Visual effect is CSS-only: `.inchi-layer.active .inchi-subtoken:hover` → `background: var(--bg-canvas)` + `box-shadow: inset 0 0 0 1.5px currentColor` at `120ms ease`.
- No React state reads `subHover` in Phase 3; the store write is wired now so Phase 4 can act on it without adding event handlers retrospectively. (D-07)

### Explanation card states

| State | Visual |
|-------|--------|
| Idle (no layer hovered) | Left border strip: `var(--ink-faint)` (no `--accent`); shows `DEFAULT_INFO` — tag label "Idle", title "Hover any layer", blurb text. No `.active` class on card. |
| Active (layer hovered) | CSS variable `--accent` set via inline style to `var(--c-{type})`; `.active` class on card; left border strip transitions to `--accent` color at `200ms`. Tag shows layer type label; title from `LAYER_INFO[type].title`; body from `.blurb`; reading-code block from `readingFor()`. |
| Transition | `background: 200ms` on `::before` (left border strip). No cross-fade on card body — content swaps immediately. |

### Legend card states

| State | Visual |
|-------|--------|
| Present layer row | Full opacity; key text in layer accent color; swatch in layer accent color |
| Absent layer row | `.muted` class; key and name in `var(--ink-faint)`; swatch at `opacity: 0.35` |
| Active type row | Row background set to `var(--c-{type}-bg)` via inline style |
| Tooltip trigger | CSS-only: `.legend-row:hover .legend-tip` → `opacity: 1`, `transform: translateY(-50%) translateX(0)` at `160ms ease` |
| Tooltip default (hidden) | `opacity: 0`, `transform: translateY(-50%) translateX(4px)`, `pointer-events: none` |

The 4px horizontal transform on tooltip entry/exit is the "slide" effect from the design handoff. No React state is needed — pure CSS hover.

### `data-layer` attribute strategy for CSS descendant rules

The `.inchi-layer` span must carry `data-layer={layer.type}` as a real HTML attribute. Inside the CSS module file, attribute selectors are written as:
```css
[data-layer="formula"] .elC { color: var(--c-el-C); }
```
This preserves the CSS specificity and descendant structure from the handoff without requiring global styles for per-element coloring.

## Implementation Notes for Executor

1. **`data-layer` attribute selectors in CSS Modules:** Write CSS rules as `[data-layer="formula"] .elC { ... }` rather than relying on global class names. Vite CSS Modules scope class names but not attribute selectors, so `[data-layer]` rules work correctly without `:global`.

2. **`--accent` CSS custom property:** Set on the explanation card wrapper via `style={{ '--accent': 'var(--c-conn)' }}` etc. TypeScript requires `style` to be typed as `React.CSSProperties` with the custom property cast: `style={{ '--accent': accentVar } as React.CSSProperties}`.

3. **`dangerouslySetInnerHTML` in reading-code block:** Permitted per D-09. The HTML string is generated by `readingFor()` from parsed InChI data. The only HTML it emits is `<b>` tags and `<span style="...">` — no user input reaches it.

4. **`swatchVar()` helper:** Maps `type → CSS token suffix`: `c → conn`, `h → hydro`, `q → charge`, `p → proton`, `i → isotope`, `b/t/m/s → stereo`, others return type unchanged. Port verbatim to `layerInfo.ts`.

5. **Legend is CSS-only hover:** The `.legend-tip` tooltip visibility is driven purely by `:hover` on `.legend-row`. No `useState` or React event handlers needed for the tooltip. This is final — do not convert to React-managed state.

6. **Store shape change:** `atomElements: Record<number, string>` must be added to `InchiState` in `store.ts` before Phase 3 components compile. The field is 1-based (`{ 1: 'C', 2: 'C', ... }`). `setInchiData` gains a fourth argument. `App.tsx` passes `result.atomElements` from `parseInchiWithAux`.

7. **`atomElements` fallback in `atomLabel`:** `atomElements[canon] ?? '#' + canon`. The `#` prefix is intentional — it signals a parsing gap without throwing.

8. **InChI display `word-break: break-all`:** Required. InChI strings have no natural word-break points and will overflow without it.

9. **Transition values — do not change:**
   - Layer active/dim: `transition: background 160ms ease, color 160ms ease` (on `.inchi-layer`)
   - Card accent border: `transition: background 200ms` (on `.card::before`)
   - Sub-token ring: `transition: background 120ms, box-shadow 120ms` (on `.inchi-subtoken`)
   - Legend tooltip: `transition: opacity 160ms ease, transform 160ms ease` (on `.legend-tip`)

10. **Legend tooltip arrow:** Pure CSS `::after` pseudo-element pointing right: `border: 6px solid transparent; border-left-color: var(--ink)`. Positioned at `left: 100%; top: 50%; transform: translateY(-50%)`.
